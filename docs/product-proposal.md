# Product Proposal: SciVerify Agent

### 1. Обоснование идеи
Анализ научных статей требует высокой точности. Обычный RAG часто ошибается. Мы используем агентную петлю (Loop), где один агент критикует другого, опираясь на первоисточник в формате Markdown (после парсинга Docling).

### 2. Цель и метрики
*   **Цель:** Обеспечить верифицируемый анализ публикаций с минимизацией галлюцинаций.
*   **Метрики:**
    *   **Success Rate:** Доля ответов, прошедших верификацию критиком без правок.
    *   **Retrieval Recall:** Качество подбора статей из 5 различных индексов Qdrant.
    *   **Traceability:** 100% покрытие цепочки рассуждений в LangFuse.

### 3. Сценарии и Edge-кейсы
*   **Сценарий:** Сравнение результатов моделей (например, точность BioBERT vs BERT) на основе таблиц из PDF.
*   **Edge-кейс:** Статья содержит противоречивые данные в тексте и таблице (агенты должны подсветить конфликт).
*   **Edge-кейс:** Запрос на тему, по которой в базе нет статей (агент должен корректно отказать, а не выдумывать).

### 4. Архитектурный набросок
Система построена на вложенных графах:
1.  **Input Graph:** Safeguard -> Translation.
2.  **Retrieval Graph:** HyDE -> Multi-index Search (via MCP).
3.  **Adversarial Graph:** Debater A (Writer) <-> Debater B (Critic).
4.  **Final Graph:** DeepAgent (Judge) выносит решение.

**Data Flow:** Запрос -> Векторизация -> Гибридный поиск -> Реранкинг -> Дебаты -> Финальный ответ.

# Governance: Risks, Security & Infrastructure

## 1. Risk Register (Расширенный)

| Риск | Вероятность | Влияние | Детекция | Защита | Остаточный риск |
|------|-------------|---------|----------|---------|-----------------|

| **Qdrant недоступен** | Низкая | Высокое | Ошибка соединения, таймаут | Retry, кэширование частых запросов, graceful degradation | Средний (нужен мониторинг) |
| **Превышение контекста LLM** | Средняя | Среднее | Проверка длины текста до вызова | Динамический чанкинг, truncation с предупреждением | Низкий |
| **Галлюцинация фактов** | Высокое | Высокое | Различие в ответах Debater A и B | Механизм дебатов, проверка по источнику | Средний (нужны метрики) |
| **Prompt Injection** | Средняя | Высокое | Узел Safeguard (LLM классификация) | Фильтрация на входе, эскапирование спецсимволов | Низкий |

## 2. SLO (Service Level Objectives)

| Метрика | Цель | Метод измерения |
|---------|------|-----------------|
| **p95 latency (полный цикл)** | < 30 сек | LangFuse трейсы |
| **p95 latency (retrieval)** | < 5 сек | MCP сервер метрики |
| **Success Rate** | > 95% | Доля успешных запросов |
| **Hallucination Rate** | < 5% | Ручная выборочная проверка |
| **API Availability** | 99.5% | Uptime мониторинг |

## 3. Политики Retry и Fallback

### LLM Calls
- **Retry**: 3 попытки, exponential backoff (1s, 2s, 4s)
- **Timeout**: 60 секунд
- **Fallback**: Сообщение "Сервис временно недоступен. Попробуйте позже."

### Qdrant Calls
- **Retry**: 2 попытки, фиксированная задержка 1s
- **Timeout**: 10 секунд
- **Fallback**: Возврат пустого результата с пометкой `error: qdrant_unavailable`

### MCP Server
- **Retry**: 2 попытки
- **Timeout**: 5 секунд
- **Fallback**: Поиск по следующей коллекции

## 4. Мониторинг и Логирование

### LangFuse Трейсы
- Каждый запрос пользователя → отдельный trace
- Каждый LLM вызов → span с model, tokens, latency
- Каждый retrieval → span с collection, query, results_count

### Метрики для алертинга
- Error rate > 5% за 5 минут → PagerDuty
- p95 latency > 45 сек → уведомление в Slack
- LLM token usage > budget → алерт

### Логи
- Уровень: INFO для production, DEBUG для dev
- Структурированные логи (JSON) для всех сервисов
- Ротация логов: 30 дней retention

## 5. Безопасность

### Защита от Injection
- Safeguard модель на входе
- Экранирование спецсимволов в промптах
- Белый список допустимых операций для MCP tools

### Работа с данными
- **Персональные данные**: не хранятся, запросы анонимизируются перед логированием
- **Секреты**: только через .env, не логируются
- **Артефакты**: PDF статьи хранятся локально, не экспортируются

## 7. Disaster Recovery

### Сценарии и планы
1. **Потеря Qdrant данных**: Реиндексация из `data/arxiv_papers/` (запуск `scripts/ingest.py`)
2. **Сбой LLM провайдера**: Переключение на backup модель (например, Anthropic Claude)
3. **Коррупция состояния агента**: Перезапуск сервиса, восстановление из последнего checkpoint

### Backup
- Ежедневный бэкап Qdrant коллекций
- Исходные PDF хранятся в S3 (планируется)
- Конфигурации в Git
